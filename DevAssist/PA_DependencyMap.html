Vertical Slice Scope
What a vertical slice is: A short, complete, shippable slice of the game that proves the core loop works end-to-end. It is not a demo of every feature — it is the most efficient possible proof that the game is fun. Everything in scope must be fully playable and polished. Everything out of scope is explicitly deferred.
✓ In Scope — Must Ship in VS
✓One complete arena encounter1 arena level, playable start to finish with waves and a clear win condition
✓All 3 weapons functionalPistol, Shotgun, Assault Rifle with correct fire behaviour, ammo, and reload
✓Rage System — full loopWP hits → meter builds → Enraged State → all 4 abilities active → state ends
✓1 equipped Special SkillAny one skill fully working. Skill swap at save point included.
✓Grapple + Dash movementGrapple anchors placed in arena; Dash rage-gated
✓Satchel ChargeThrow, fuse, blast jump, crowd control, cooldown
✓Parry systemNormal + Perfect parry, parry window, rage-widened window
✓All 5 enemy typesBrute, Flying Eye, Spawner, FodderMelee, FodderRanged each with functional AI
✓All enemy animationsLocomotion, attack, stun, death (ragdoll) for each enemy
✓Player animationsLocomotion, grapple, dash, parry, death montages
✓Weak point systemVisual WP actors on sockets, hit detection, armour break on Brute
✓Save point + skill swap UIPlaced in arena, functional skill selection menu
✓Core HUDHealth bar, rage meter, ammo display, skill slot display, weapon wheel
✓Death screenPlayer death → screen → respawn (reload level)
✓Pause menuPause/resume/quit functional
✓Footstep audio + VFXAt minimum 2 surface types (concrete + metal)
✓Combat audioAll weapon fire, impacts, enemy alerts, satchel explosion
✓PP_Enrage + PP_HitFlashPost process effects on both rage and damage
✓Camera shake on hitFeedback on player taking damage
✗ Out of Scope — Explicitly Deferred
✗Story / narrativeNo cutscenes, no dialogue, no Act 1 story beats in the VS
✗Multiple levels / Act structureThe VS is one arena. Level streaming and the 4-era hub are post-VS.
✗Multiple Special SkillsOnly 1 skill needs to work. The full DT_Skills registry is post-VS.
✗Enemy C (Spawner) full budget tuningSpawner functions but spawn budget numbers are placeholder
✗Full IK (foot IK)Foot IK is a polish feature. VS uses base locomotion without adaptive foot placement.
✗Main menu level (L_MainMenu)Game starts directly in arena for VS
✗Loading screenNo level transitions in VS; WBP_LoadingScreen deferred
✗Settings menuPause menu has Quit only; Settings is a stub
✗Audio volume settingsSound Classes set up but no in-game volume sliders
✗Music systemSC_Music exists but no music tracks in VS
✗Multiple skill typesPost-VS expansion
✗Post-process LUT per actSingle LUT or no LUT in VS
✗Gamepad full supportIMC_ keyboard/mouse only for VS; gamepad bindings added post-VS
✗Hit reaction directional variantsSingle hit react montage in VS; 4-directional variants post-VS
✗Dissolve death FXRagdoll + destroy without dissolve in VS; MF_Dissolve post-VS
