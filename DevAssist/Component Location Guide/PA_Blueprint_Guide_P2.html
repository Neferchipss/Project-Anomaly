§01
Remaining Structs + DataTables
Content Browser location
📁Content/ProjectAnomaly/Data/📁Content/ProjectAnomaly/Data/Structs/
All structs live in Data/Structs/. All DataTables live in Data/. These are pure data assets with no Blueprint graph — create them by right-clicking in the correct folder, picking the type, and filling in the rows.
CREATE
All FS_ structs in Data/Structs/

Navigate to Data/Structs/. FS_WeaponRow: MaxAmmo (int), BaseDamage (float), ReloadTime (float), WeaponClass (soft class). FS_EnemyRow: EnemyClass (soft class), MaxHealth, MaxArmour (float), WeakPointCount (int), RagePerWeakPointHit (float), bCanBeStunned (bool), StunThreshold (float), bFiresProjectiles (bool), SightRadius (float). FS_SkillRow and FS_WaveRow: see TDD §Structs.

CREATE
All DataTables in Data/

Navigate to Data/. Right-click → Miscellaneous → Data Table. DT_Weapons (FS_WeaponRow): rows Pistol, Shotgun, AssaultRifle. DT_Enemies (FS_EnemyRow): rows Brute, FlyingEye, Spawner, FodderMelee, FodderRanged. DT_Skills (FS_SkillRow): row BlastWave. DT_ArenaWaves (FS_WaveRow): 3 wave rows. Row names are case-sensitive strings — a typo causes a silent GetDataTableRow failure at runtime.

§02
BP_Weapon_Shotgun + BP_Weapon_AssaultRifle
Content Browser location
📁Content/ProjectAnomaly/Weapons/AssaultRifle/📁Content/ProjectAnomaly/Weapons/Shotgun/
BP_Weapon_Shotgun lives in Weapons/Shotgun/. BP_Weapon_AssaultRifle lives in Weapons/AssaultRifle/. Both are children of BP_WeaponBase (which lives in Weapons/Base/). Navigate to the correct subfolder before creating each child BP.
CREATE
BP_Weapon_Shotgun in Weapons/Shotgun/

Navigate to Weapons/Shotgun/. Blueprint Class → parent=BP_WeaponBase → BP_Weapon_Shotgun. Override Fire(): Branch ammo>0 → For Loop 0–7 (8 pellets) → Random Unit Vector In Cone (5°) → Line Trace By Channel → Branch bBlockingHit → BPI_Damageable.ReceiveDamage. Penetration: if cast to Fodder succeeds and PenetrationCount<3, re-trace from hit location. Completed exec: SET ammo−1 → Call OnFired.

CREATE
BP_Weapon_AssaultRifle in Weapons/AssaultRifle/

Navigate to Weapons/AssaultRifle/. Override Fire(): Branch NOT bIsFiring → bIsFiring=true → CharMov.MaxWalkSpeed=200 → Set Timer "DoFireTick" CurrentFireInterval (looping=false). Custom Event DoFireTick: Branch bIsFiring AND ammo>0 → Line Trace → FInterp(Current, Min, WindUpRate) → SET CurrentFireInterval → re-schedule. Else: StopFire (reset all, Clear Timer, MaxWalkSpeed=600).

§03
BP_Projectile_Base + BP_Projectile_Enemy
Content Browser location
📁Content/ProjectAnomaly/Combat/
Both live in Combat/. BP_Projectile_Base is the abstract parent; BP_Projectile_Enemy is the child that adds parry window timing. Navigate to Combat/ before creating either.
CREATE
BP_Projectile_Base in Combat/ (abstract parent)

Navigate to Combat/. Blueprint Class → parent=Actor → check Abstract → BP_Projectile_Base. Add Sphere Collision (r=15), ProjectileMovementComponent (speed=2000). Variables: Damage (float=20), bCanBeParried (bool=false), Instigator_Enemy (BP_EnemyBase ref). On Component Hit: BPI_Damageable.ReceiveDamage on Other → Destroy Actor. Implement BPI_Parryable: CanBeParried returns bCanBeParried; OnParried→Destroy; OnPerfectParried→Destroy+AddStunDamage(999).

CREATE
BP_Projectile_Enemy in Combat/ (child with parry window)

Still in Combat/. Blueprint Class → parent=BP_Projectile_Base → BP_Projectile_Enemy. BeginPlay: Set Timer "OpenParryWindow" 0.3s. OpenParryWindow event: SET bCanBeParried=true → Set Scalar Parameter "GlowIntensity"=1.0 on mesh material → Set Timer "CloseParryWindow" 0.6s. CloseParryWindow: SET bCanBeParried=false → GlowIntensity=0.

§04
BP_SkillBase + BP_Skill_BlastWave
Content Browser location
📁Content/ProjectAnomaly/Skills/📁Content/ProjectAnomaly/Skills/Base/
BP_SkillBase and DA_SkillData live in Skills/Base/. Concrete skill BPs (like BP_Skill_BlastWave) live directly in Skills/. Adding a new skill = create a child BP in Skills/ and add a DT_Skills row — no other system changes needed.
CREATE
BP_SkillBase in Skills/Base/ — DA_SkillData in Skills/Base/

Navigate to Skills/Base/. Blueprint Class → parent=Actor → check Abstract → BP_SkillBase. Variables: SkillDataRowName (Name), OwnerPawn (Pawn), Duration (float). Implement BPI_SkillEquippable: Execute (empty), GetSkillData (returns DA ref). Also create DA_SkillData as a Data Asset (right-click → Miscellaneous → Data Asset).

CREATE
BP_Skill_BlastWave in Skills/

Navigate to Skills/. Blueprint Class → parent=BP_SkillBase → BP_Skill_BlastWave. Override Execute(Instigator Pawn): Get All Actors In Radius (InstigatorLocation, r=600) → For Each → BPI_Damageable.ReceiveDamage(BaseDamage=150) → Spawn NS_SatchelExplosion at InstigatorLocation → Destroy Actor (self).

§05
Enemy Child BPs — BP_EnemyBase + all 5 types
Content Browser location
📁Content/ProjectAnomaly/Enemies/Base/📁Content/ProjectAnomaly/Enemies/Brute/📁Content/ProjectAnomaly/Enemies/FlyingEye/📁Content/ProjectAnomaly/Enemies/Fodder/📁Content/ProjectAnomaly/Enemies/Spawner/
BP_EnemyBase lives in Enemies/Base/. Each enemy child BP lives in its own subfolder: BP_Enemy_Brute in Enemies/Brute/, BP_Enemy_FlyingEye in Enemies/FlyingEye/, BP_Enemy_Spawner in Enemies/Spawner/, BP_Enemy_FodderMelee and BP_Enemy_FodderRanged both in Enemies/Fodder/.
CREATE
BP_EnemyBase in Enemies/Base/ (abstract parent)

Navigate to Enemies/Base/. Blueprint Class → parent=Character → check Abstract → BP_EnemyBase. Add components: AC_EnemyHealthComponent, AC_WeakPointComponent, AC_StunComponent, AIPerceptionComponent (Sight config). Class Settings → implement BPI_Damageable (routes to AC_EnemyHealthComponent.TakeDamage) and BPI_WeakPointTarget. BeginPlay: Get Data Table Row (DT_Enemies, RowName = variable set per child) → Break → push values to all 3 components.

CREATE
Each child BP in its own subfolder

Enemies/Brute/: Blueprint Class → parent=BP_EnemyBase → BP_Enemy_Brute. Set EnemiesTableRowName="Brute", WPSocketNames=["WP_Chest","WP_ShoulderL","WP_ShoulderR"], AI Controller Class=AIC_Brute.
Enemies/FlyingEye/: BP_Enemy_FlyingEye. Row="FlyingEye", WPSocketNames=["WP_Eye","WP_Wing"], AIC=AIC_FlyingEye.
Enemies/Spawner/: BP_Enemy_Spawner. Row="Spawner", add AC_SpawnDirectorComponent in components panel, AIC=AIC_Spawner.
Enemies/Fodder/: BP_Enemy_FodderMelee and BP_Enemy_FodderRanged. Rows match, WPSocketNames=[] (empty), AIC=AIC_Fodder for both.

§06
EQS — Environment Query Systems
Content Browser location
📁Content/ProjectAnomaly/Enemies/FlyingEye/📁Content/ProjectAnomaly/Enemies/Spawner/
EQS assets live in the enemy's own subfolder: EQS_FlyingEye_Strafe in Enemies/FlyingEye/, EQS_Spawner_Retreat in Enemies/Spawner/. Right-click in the correct folder → Artificial Intelligence → Environment Query.
CREATE
EQS_FlyingEye_Strafe in Enemies/FlyingEye/

Navigate to Enemies/FlyingEye/. Right-click → Artificial Intelligence → Environment Query → EQS_FlyingEye_Strafe. Inside: Generator = Points Circle (Querier Context, Radius=400, NumPoints=16). Add Distance Test to Player (preferred range 300–500). This query finds lateral positions for the eye to strafe to.

CREATE
EQS_Spawner_Retreat in Enemies/Spawner/

Navigate to Enemies/Spawner/. Environment Query → EQS_Spawner_Retreat. Generator = Grid (2000×2000, spacing=200). Add Distance Test to Player: filter out points closer than 600 units, score by maximum distance. This pushes the Spawner to the far edge of the arena.

§07
Shared BT Task Nodes
Content Browser location
📁Content/ProjectAnomaly/Enemies/Shared_AI/
All shared BT nodes live in Enemies/Shared_AI/. Navigate there before creating any of them. Right-click → Artificial Intelligence → Behavior Tree Task (for BTT_), Behavior Tree Service (for BTS_), or Behavior Tree Decorator (for BTD_).
CREATE
All shared nodes in Enemies/Shared_AI/

Navigate to Enemies/Shared_AI/. Create all of these here:
BTT_FindPlayerLocation: Receive Execute AI → Get Player Character → Get Actor Location → Set BB Value as Vector "TargetLocation" → Finish Execute Success=true.
BTT_AttackMelee (CRITICAL — damage on notify): Play Anim Montage → GET Anim Instance → Bind to On Montage Notify Begin → OnAttackNotify(Name): Branch Name=="AttackHit" → BPI_Damageable on BB.TargetActor → Unbind → Finish Execute.
BTT_FireProjectile: Normalise(TargetLoc−PawnLoc) → Spawn BP_Projectile_Enemy (offset 80u) → SET Instigator_Enemy → Finish Execute.
BTT_SpawnFodder: Cast pawn → GET AC_SpawnDirectorComponent → TrySpawnFodder() → Finish Execute.

§08
BP_ArenaManager + BP_EnemySpawner
Content Browser location
📁Content/ProjectAnomaly/Arena/
Both live in Arena/. BP_ArenaManager orchestrates wave spawning. BP_EnemySpawner is a simple placed marker actor — the ArenaManager picks randomly from the ones placed in the level.
CREATE
BP_ArenaManager in Arena/

Navigate to Arena/. Blueprint Class → parent=Actor → BP_ArenaManager. Variables: ActiveEnemies (BP_EnemyBase array), bArenaActive (bool). StartWave(): bArenaActive=true → GET GS → Cast BP_GS_Anomaly → SET bArenaActive → GET all DT_ArenaWaves rows → For Each → Set Timer(row.SpawnDelay) → SpawnBatch(row). SpawnBatch(FS_WaveRow): For Loop SpawnCount → Get All Actors Of Class BP_EnemySpawner → random one → Spawn Actor(row.EnemyClass) → bind HealthComp.OnDeath → OnEnemyDied → ADD to ActiveEnemies. OnEnemyDied: REMOVE → Branch Num==0 → Call OnWaveComplete.

CREATE
BP_EnemySpawner in Arena/ — then place instances in level

Still in Arena/. Blueprint Class → parent=Actor → BP_EnemySpawner. Add a visible mesh or billboard component so it's visible in the editor. No Blueprint graph needed — this is a pure position marker. Place a minimum of 3 instances in the arena level at varied positions.

§09
BP_GI_Anomaly + BP_GS_Anomaly + save system
Content Browser location
📁Content/ProjectAnomaly/Core/GameInstances/📁Content/ProjectAnomaly/Core/GameModes/📁Content/ProjectAnomaly/Core/SaveGame/
BP_GI_Anomaly lives in Core/GameInstances/. BP_GS_Anomaly lives in Core/GameModes/. BP_Save_Anomaly lives in Core/SaveGame/. The Game Instance persists across level loads — it's the only place safe to store data that must survive an Open Level call.
CREATE
BP_Save_Anomaly in Core/SaveGame/

Navigate to Core/SaveGame/. Blueprint Class → parent=SaveGame → BP_Save_Anomaly. Add variable: EquippedSkillRowName (Name). This is the only data that needs to persist — the equipped skill name is enough to reconstruct state on load.

WIRE
BP_GI_Anomaly in Core/GameInstances/ — Event Init + SaveGame function

Variables: SavedEquippedSkillName (Name), bHasSaveData (bool). Event Init: Does Save Game Exist("SaveSlot_01") → Branch true → Load Game from Slot → Cast BP_Save_Anomaly → SET SavedEquippedSkillName + bHasSaveData=true. Function SaveGame: Create Save Game Object (BP_Save_Anomaly) → SET .EquippedSkillRowName=SavedEquippedSkillName → Save Game to Slot "SaveSlot_01". Set as Game Instance Class in Project Settings.

§11
Remaining HUD Widgets — health, rage, ammo, skill
Content Browser location
📁Content/ProjectAnomaly/UI/
All HUD sub-widgets live in UI/. They are created inside WBP_HUD's canvas — not as standalone screens. Bindings should reference the player character's components, not hardcoded values.
CREATE
All sub-widgets in UI/ — add to WBP_HUD canvas

Navigate to UI/. Create each Widget Blueprint. WBP_HealthBar: Progress Bar, Bind Percent = CurrentHealth ÷ MaxHealth, Bind Fill Color (green→red lerp below 0.2). WBP_RageMeter: Progress Bar, Bind Percent = BPI_Rageable.GetRagePercent. WBP_AmmoDisplay: Text Block, Bind Text = WeaponSlots[ActiveWeaponIndex].CurrentAmmo → To Text (Int). WBP_SkillSlotDisplay: Image + Border (glow overlay), binds to OnSkillEquipped and OnEnragedStateBegin/End dispatchers. Open WBP_HUD and add each sub-widget to its container.

§12
DA_WeaponData
Content Browser location
📁Content/ProjectAnomaly/Weapons/Base/
DA_WeaponData lives in Weapons/Base/ alongside BP_WeaponBase. Create it as a Data Asset (right-click → Miscellaneous → Data Asset → pick the DA_WeaponData class). One DA per weapon is normal — or you can use DT_Weapons for all stats and leave DA_WeaponData for assets only (sound refs, mesh refs).
CREATE
DA_WeaponData in Weapons/Base/

Navigate to Weapons/Base/. Right-click → Miscellaneous → Data Asset → pick DA_WeaponData as the class → name it DA_WeaponData (or one per weapon if using per-weapon assets). Assign it in each weapon BP's Class Defaults under the WeaponData variable. This is where damage, ammo counts, sound cues, and mesh references are set per weapon.
