§01
Enhanced Input — IMC_Default setup
Content Browser location
📁Content/ProjectAnomaly/Input/
All input assets live in Input/. This section covers the critical step that Part 1 omitted: registering the IMC_ in BP_PlayerController BeginPlay. Without Add Mapping Context, every input action is permanently silent.
CRITICAL
Add Mapping Context in BP_PlayerController BeginPlay

Open BP_PlayerController (in Player/Character/). In Event BeginPlay: Get Enhanced Input Local Player Subsystem (pass Self) → Add Mapping Context (IMC_Default, Priority=0). This must be the first thing in BeginPlay — before any other input-dependent setup. Then continue with Create Widget WBP_HUD → Add to Viewport.

SETUP
All IA_ assets in Input/ — Value Types

Navigate to Input/. Create all 10 Input Action assets. Right-click → Input → Input Action. Value Types must be exact: IA_Fire, IA_Parry, IA_Grapple, IA_Dash, IA_Utility, IA_Skill, IA_WeaponWheel, IA_Jump = Digital (bool). IA_Move, IA_Look = Axis2D (Vector2D). Create IMC_Default (Input → Input Mapping Context) in the same Input/ folder and bind all 10 keys.

§02
ABP_Player — Animation Blueprint
Content Browser location
📁Content/ProjectAnomaly/Player/Animation/📁Content/ProjectAnomaly/Player/Animation/BlendSpaces/
ABP_Player lives in Player/Animation/. BS_Player_Locomotion (the Blend Space it uses) lives in Player/Animation/BlendSpaces/. Create the ABP_ first, then create the Blend Space and wire it into the AnimGraph.
CREATE
ABP_Player in Player/Animation/ — assign to BP_PlayerCharacter

Navigate to Player/Animation/. Right-click → Animation → Animation Blueprint → select SK_Player skeleton → name ABP_Player. Open BP_PlayerCharacter → select Mesh component → Details → Anim Class → assign ABP_Player. This makes the ABP_ run on the player mesh.

CREATE
BS_Player_Locomotion in Player/Animation/BlendSpaces/

Navigate to Player/Animation/BlendSpaces/. Right-click → Animation → Blend Space → select SK_Player skeleton → BS_Player_Locomotion. Open it: Horizontal Axis = Speed (0–600), Vertical Axis = Direction (−180 to 180). Add sample points: Idle(0,0), WalkFwd(300,0), RunFwd(600,0), WalkL/R/Back. In ABP_Player AnimGraph: Blend Space Player node (inputs: Speed, Direction variables) → Slot "DefaultSlot" → Output Pose.

§03
ABP_Enemy + AIC→ABP push pattern
Content Browser location
📁Content/ProjectAnomaly/Enemies/Brute/Animations/📁Content/ProjectAnomaly/Enemies/FlyingEye/Animations/📁Content/ProjectAnomaly/Enemies/Fodder/Animations/📁Content/ProjectAnomaly/Enemies/Spawner/Animations/
Each enemy's ABP_ lives in its own Animations/ subfolder: ABP_Brute in Enemies/Brute/Animations/, ABP_FlyingEye in Enemies/FlyingEye/Animations/, ABP_Spawner in Enemies/Spawner/Animations/, ABP_Fodder in Enemies/Fodder/Animations/ (shared for both fodder types).
CREATE
Each ABP_ in its enemy's Animations/ subfolder

Navigate to each enemy's Animations/ folder. Right-click → Animation → Animation Blueprint → select that enemy's SK_ skeleton. ABP_Brute: vars Speed, bIsStunned, bIsDead, bArmourBroken. AnimGraph State Machine: Locomotion (BS_Brute_Locomotion), Stun (AM_Brute_Stun loop), Death (AM_Brute_Death, terminal). Same pattern for FlyingEye, Spawner, Fodder — vars and states match their enemy type.

WIRE
AIC→ABP push: wired inside each AIC_ in the enemy's subfolder

Open each AIC_ (e.g. AIC_Brute in Enemies/Brute/). In OnPossess, after Run Behavior Tree: GET Mesh → GET Anim Instance → Cast to ABP_Brute → store ref. Then bind component dispatchers: HealthComp.OnDeath → SET ABP_Brute.bIsDead=true. StunComp.OnStunBegin → SET bIsStunned=true. StunComp.OnStunEnd → SET bIsStunned=false. HealthComp.OnArmourBroken → SET bArmourBroken=true. The AIC_ pushes state; the ABP_ reads it.

§04
Anim Notifies — AN_ and ANS_
Content Browser location
📁Content/ProjectAnomaly/Enemies/Brute/Animations/Notifies/📁Content/ProjectAnomaly/Player/Animation/Notifies/
Anim Notify assets live in Notifies/ subfolders. Player notifies live in Player/Animation/Notifies/. Enemy notifies live in each enemy's Animations/Notifies/ subfolder (e.g. Enemies/Brute/Animations/Notifies/). Create the notify asset in the correct Notifies/ folder, then place it on the montage timeline.
CREATE
AN_Brute_AttackHit in Enemies/Brute/Animations/Notifies/

Navigate to Enemies/Brute/Animations/Notifies/. Right-click → Animation → Anim Notify → AN_Brute_AttackHit. Open AM_Brute_Attack montage (in Enemies/Brute/Animations/Montages/). In the timeline, right-click the Notifies track at the impact frame → Add Notify → AN_Brute_AttackHit. In BTT_AttackMelee: Bind to On Montage Notify Begin → filter by Name=="AttackHit" → apply damage → Unbind → Finish Execute. This is critical — damage on notify means it lands at the visual impact frame, not at montage end.

CREATE
ANS_Brute_AttackWindow in same Notifies/ folder

Navigate to Enemies/Brute/Animations/Notifies/. Right-click → Animation → Anim Notify State → ANS_Brute_AttackWindow. Implement NotifyBegin (SET bAttackWindowOpen=true) and NotifyEnd (SET false). In AM_Brute_Attack, drag the ANS_ to span the full swing duration. Unlike AN_ (a point in time), ANS_ covers a duration — useful for any window-based logic.

§05
Footstep System — AN_ notifies + DT_FootstepSurfaces
Content Browser location
📁Content/ProjectAnomaly/Data/📁Content/ProjectAnomaly/Player/Animation/Notifies/
Player footstep Anim Notifies live in Player/Animation/Notifies/. DT_FootstepSurfaces lives in Data/. The notify fires at foot-strike frame, traces down to read the Physical Material, looks up the matching sound in the DataTable, and plays it.
CREATE
AN_Player_FootstepL + AN_Player_FootstepR in Player/Animation/Notifies/

Navigate to Player/Animation/Notifies/. Right-click → Animation → Anim Notify → create both. In each, implement the Notify function: Get Socket Location ("foot_l" or "foot_r") → Line Trace By Channel (Start=socket loc, End=Start+(0,0,−20)) → Branch bHit → Break HitResult → Get Surface Type → Get Data Table Row (DT_FootstepSurfaces, RowName=surface name) → Play Sound at Location (row.FootstepSound, ImpactPoint) → Spawn NS_Footstep_Dust. Place both notifies on locomotion animation assets by opening each anim → right-click Notifies track at foot-strike frame → Add Notify → select AN_Player_FootstepL/R.

§06
Hit reactions + Death ragdoll
Content Browser location
📁Content/ProjectAnomaly/Enemies/Base/📁Content/ProjectAnomaly/Materials/Functions/
All hit reaction and ragdoll logic is wired inside BP_EnemyBase (in Enemies/Base/) — not in child BPs. MF_Dissolve (the material function driving the death dissolve) lives in Materials/Functions/. PHYS_ physics assets for each enemy live in the enemy's own subfolder.
WIRE
Hit reactions in BP_EnemyBase (Enemies/Base/)

Open BP_EnemyBase. In BeginPlay: Bind AC_EnemyHealthComponent.OnDamageTaken → OnHit_React(DamageInfo): Break → HitLocation−ActorLocation → Normalize → Dot Product with ActorForwardVector → Branch >0.5: front montage (AM_HitReact_Front, BlendIn=0.1) / <−0.5: back / else Cross Product for left/right. Montage plays into DefaultSlot and auto-blends back after ~0.3s.

WIRE
Death ragdoll in BP_EnemyBase (3-phase)

Bind OnDeath → OnDeath_Ragdoll: Play AM_Death → Set Timer "ActivateRagdoll" 0.2s. ActivateRagdoll: Get Mesh → Set All Bodies Simulate Physics=true → Set Collision Profile Name="Ragdoll" → Get Capsule → Set Collision Enabled=No Collision → Add Impulse(Velocity×1.5, bVelChange=true) → Set Timer "DissolveAndDestroy" 5.0s. DissolveAndDestroy: Create Dynamic Material Instance → Play Timeline Dissolve (0→1, 1.5s) → Set Scalar "DissolveAmount" → Timeline Finished → Destroy Actor. The PHYS_ asset for each enemy must exist in the enemy's subfolder (e.g. PHYS_Enemy_Brute in Enemies/Brute/).

§07
IK Rig + Foot IK
Content Browser location
📁Content/ProjectAnomaly/Player/Mesh/
IKR_Player lives in Player/Mesh/ alongside SK_Player. Two Bone IK nodes are added inside ABP_Player's AnimGraph (in Player/Animation/). No separate folder needed for the IK nodes — they live inside the ABP_.
CREATE
IKR_Player in Player/Mesh/

Navigate to Player/Mesh/. Right-click → Animation → IK Rig → select SK_Player → IKR_Player. Open it → assign SK_Player as Preview Mesh → add IK Chains for each leg (thigh to foot bone). Then open ABP_Player AnimGraph: after the DefaultSlot node, add Two Bone IK nodes for each foot. In ABP_Player Event Graph: add foot trace logic each tick (Get Socket Location → Line Trace down 20 units → write IKFootL/R vector variables). The Two Bone IK nodes read these variables.

§08
Post Process Effects — PP_Enrage + PP_HitFlash
Content Browser location
📁Content/ProjectAnomaly/Materials/PostProcess/
Both PP materials live in Materials/PostProcess/. The PostProcessVolume is placed in the level (not in the Content Browser). The PostProcessComponent that drives them is added to BP_PlayerCharacter in Player/Character/.
CREATE
PP_Enrage + PP_HitFlash in Materials/PostProcess/

Navigate to Materials/PostProcess/. Right-click → Material → PP_Enrage. Open it → Details panel → Material Domain = Post Process. Add Scalar Parameter node named "EnrageIntensity" (default=0). Build effect: Lerp between neutral and red-tinted scene colour using EnrageIntensity. Repeat for PP_HitFlash with "HitFlashIntensity" and a white flash effect. Place a PostProcessVolume in the level → Infinite Extent=true → add both PP_ materials to its Materials array.

§09
Sound Classes + Sound Mixes
Content Browser location
📁Content/ProjectAnomaly/Audio/SoundClasses/📁Content/ProjectAnomaly/Audio/SoundMixes/
Sound Classes live in Audio/SoundClasses/. Sound Mixes live in Audio/SoundMixes/. Right-click in the correct folder → Sounds → Sound Class (or Sound Mix). Navigate there before creating — they look similar in the create dialog.
CREATE
SC_Master / SC_SFX / SC_UI in Audio/SoundClasses/

Navigate to Audio/SoundClasses/. Right-click → Sounds → Sound Class. Create SC_Master (no parent), SC_SFX (parent=SC_Master), SC_UI (parent=SC_Master). Set the parent by opening each asset → Details panel → Parent Class. Assign SC_SFX to all in-game sound cue assets by opening each → Details → Sound Class.

CREATE
SM_Gameplay + SM_Pause in Audio/SoundMixes/

Navigate to Audio/SoundMixes/. Right-click → Sounds → Sound Mix. SM_Gameplay: leave empty. SM_Pause: Sound Class Effects array → add entry: Sound Class=SC_SFX, Volume Adjuster=0.1, Pitch Adjuster=1.0. In BP_PlayerController pause toggle: Push Sound Mix Modifier(SM_Pause) on open, Pop Sound Mix Modifier(SM_Pause) on close.

§10
WBP_PauseMenu + WBP_DeathScreen + WBP_LoadingScreen
Content Browser location
📁Content/ProjectAnomaly/UI/
All three live in UI/. Create them as User Interface → Widget Blueprint. ZOrder hierarchy matters: WBP_PauseMenu=10, WBP_DeathScreen=20, WBP_LoadingScreen=100. Higher ZOrder appears on top.
CREATE
WBP_PauseMenu in UI/ — toggle wired in BP_PlayerController

Navigate to UI/. User Interface → Widget Blueprint → WBP_PauseMenu. Layout: Resume button + Quit button. Wiring lives in BP_PlayerController (Player/Character/): var PauseMenuWidget. IA_Pause Started: Branch Is Valid(PauseMenuWidget) → [TRUE: Remove from Parent, null, Set Game Paused=false, Input Mode Game Only, Hide Cursor, Pop SM_Pause] / [FALSE: Create Widget, SET ref, Add to Viewport ZOrder=10, Set Game Paused=true, Input Mode UI Only, Show Cursor, Push SM_Pause].

CREATE
WBP_DeathScreen in UI/ — triggered from BP_PlayerCharacter

Still in UI/. Widget Blueprint → WBP_DeathScreen. Add UMG animation "FadeIn" (Opacity 0→1 over 0.5s), play in Event Construct. Respawn button: Open Level by Name (Get Current Level Name). Main Menu: Open Level "L_MainMenu". Wiring lives in BP_PlayerCharacter (Player/Character/): bind OnDeath → Play AM_Player_Death → Disable Input → Set Timer "ShowDeathScreen" 2.5s. ShowDeathScreen: Create Widget WBP_DeathScreen → Add to Viewport ZOrder=20 → Set Input Mode UI Only.
